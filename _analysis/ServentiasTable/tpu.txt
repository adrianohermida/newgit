🧠 PROMPT APRIMORADO – JUÍZO CNJ
Módulo: DataJud → CNJParse → Camada Jurisdicional
CONTEXTO GERAL

Você está evoluindo o CNJParse, no contexto do DataJud, para estruturar corretamente a dimensão jurisdicional do Judiciário brasileiro.

O Juízo CNJ representa a autoridade jurisdicional competente (órgão julgador), sendo conceitualmente distinto de:

Serventia (estrutura administrativa)

Foro / Comarca (estrutura territorial)

Tribunal (estrutura institucional)

O sistema deve refletir essa separação de forma explícita, auditável e reutilizável.

🎯 OBJETIVO DO MÓDULO JuizoCNJ

O módulo JuizoCNJ deve:

Representar corretamente o órgão jurisdicional

Resolver ambiguidades do DataJud (ex.: nomes genéricos)

Servir de base para:

competência

estatística judicial

distribuição de processos

regras de prazo por grau de jurisdição

Conectar processos à estrutura correta do Judiciário

Permitir importação oficial e gestão manual

🧩 CONCEITO JURÍDICO FUNDAMENTAL

Juízo ≠ Serventia

Juízo → função jurisdicional (quem julga)

Serventia → unidade administrativa (onde tramita)

📌 Um Juízo pode existir sem serventia própria
📌 Uma Serventia sempre deve apontar para um Juízo

1️⃣ Schema — JuizoCNJ (ENTIDADE NÚCLEO)
Finalidade

Normalizar e identificar de forma única o órgão julgador informado pelo CNJ/DataJud.

📐 Schema lógico obrigatório
JuizoCNJ {
  id,
  tribunal,                  // TJSP, TRF1, TRT15, STJ etc.
  uf,
  grau: primeiro | segundo | superior,
  orgao_julgador,             // Ex: 1ª Vara Cível, 3ª Câmara
  competencia,                // Cível, Criminal, Família etc.
  codigo_cnj,                 // Quando existente
  area_direito,               // civil, penal, trabalhista etc.
  serventiaId,                // FK opcional
  ativo
}

🔐 Regras estruturais

codigo_cnj não é obrigatório (nem sempre existe)

orgao_julgador não pode ser tratado como string solta

grau impacta diretamente:

prazos

recursos

estatística

ativo deve ser soft state (nunca delete físico)

🧠 PROMPT APRIMORADO – TPU AMPLIADA

A TPU (Tabela Processual Unificada) deve ser tratada como um sistema vivo de classificação jurídica, composto por Classes, Assuntos e Movimentos, cada qual com papel próprio no ciclo de vida do processo.

🎯 OBJETIVO DO APRIMORAMENTO

Classe ≠ Assunto ≠ Movimento

Assunto → tema jurídico discutido

Misturar isso quebra prazo, BI e automação.
